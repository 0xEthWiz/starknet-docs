Simplified transaction flow

The key stages of transaction lifetime are:

  1. A new transaction is sent to the mempool with a nonce

  2. The sequencer selects a valid transaction from the mempool

  3. If the transaction is valid, the execute function is called

  4. If execute is successful, the sequencer passes the transaction to the StarkNet prover
